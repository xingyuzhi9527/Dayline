# Dayline「盘」页重构计划书

## 1. 背景与判断

当前「盘」页的问题不是功能不够多，而是页面像一份功能清单：统计卡、分类列表、关键词、AI 总结、晚间复盘、导出按钮依次堆叠。用户看完之后很难得到一个明确答案：

- 今天过得怎么样？
- 哪里有明显空白？
- 什么值得保留？
- 明天应该先做哪一步？

Dayline 的核心价值不是做一个复杂报表工具，而是把用户通过「记」页留下的一句话、语音、待办、情绪、专注、消费、身体等碎片，整理成一天的判断和一篇可沉淀的笔记。

因此，「盘」页应从“统计面板”重构为“今日复盘仪式”：平时轻、晚上展开、数据真实、最后能生成 Markdown，成为以后接入 AI 的个人数据库入口。

## 2. 目标用户与核心需求

### 目标用户

1. 低成本记录型用户
   - 不想写长日记，只想随口说一句或打一行字。
   - 需要系统自动整理，而不是要求自己手动分类。

2. 复盘但怕麻烦的用户
   - 晚上愿意花 1-3 分钟看一下今天。
   - 只愿意补几句话，不想面对一堆表单。

3. 工作、学习、生活混合记录用户
   - 希望看到今天偏向哪里。
   - 希望发现空白时间、情绪波动、专注断点。

4. 未来 AI 个人数据库用户
   - 希望每天自动沉淀为结构化 Markdown。
   - 以后可以让 AI 基于历史笔记总结模式、习惯和长期趋势。

### 核心需求

- 快速知道今天的状态，而不是看一堆数字。
- 能看到记录密度和空白区域。
- 能把「记」页数据自动同步到「盘」页。
- 晚上有一个轻量复盘入口。
- 一键生成日期命名的 Markdown 笔记。
- 笔记结构长期稳定，方便以后接 AI。

## 3. 三个主页面的新职责

### 记

低摩擦输入入口。

- 语音记录
- 文字记录
- 待办记录
- 自动解析为 memo、todo、tracker、focus、expense、body、sleep、mood

用户只负责“留下内容”。

### 线

事实回看入口。

- 按时间看今天发生了什么。
- 记录和待办分列。
- 时间空白通过线段长短体现。
- 支持编辑、查看、补充。

用户在这里看“发生顺序”。

### 盘

解释与沉淀入口。

- 今天状态如何。
- 今天的节奏如何。
- 今天由什么构成。
- 今天有哪些值得注意的判断。
- 晚间补三句复盘。
- 生成一篇 Markdown 日记。

用户在这里把“一天炼成一篇笔记”。

## 4. 核心概念：复盘球

「盘」页默认不再展示一长串卡片，而是展示一个可动的球形入口。这个球是“一天的容器”。

### 默认状态

页面中心是一个会轻微呼吸的球。

球体可以表达当天状态：

- 记录越多，球面越充实。
- 有待办未完成，球体边缘出现小刻度。
- 有长时间空白，球体内部出现淡淡暗区。
- 晚上 6 点后，球体开始微微发光，提示可以复盘。
- 已完成复盘后，球体变为稳定柔和状态。

### 打开方式

用户点击球体，进入展开态。展开过程类似“打开一个容器”，但视觉不宜过度游戏化，应保持 Dayline 的安静、生活化和高级感。

建议命名：

- UI 内可叫「今日复盘」
- 代码组件可叫 `ReviewOrb` 或 `DailyReviewOrb`
- 不在界面直接使用“精灵球”这个词

### 固定打开时间

默认晚上 18:00 后进入“建议复盘”状态。

可选规则：

- 00:00-17:59：轻量收起态，强调“今天还在发生”。
- 18:00-23:59：复盘邀请态，球体微光，文案提示“今天可以收束了”。
- 已完成复盘：完成态，显示“已生成今日笔记”或“已完成复盘”。

后续可以在设置中开放复盘时间，例如 18:00、20:00、睡前。

## 5. 信息架构

### 收起态

只展示少量信息：

- 标题：今日复盘
- 日期
- 复盘球
- 一句状态摘要
- 一个主按钮：打开复盘

示例状态文案：

- “今天已经留下 12 条记录。”
- “下午有一段 2 小时空白。”
- “晚上可以整理成一篇日记。”
- “今天还没有记录，先去记一条。”

### 展开态

打开后分为六块。

#### 1. 今日状态

合并当前四宫格统计，改为一张主状态卡。

内容示例：

> 今天留下了 12 条记录，完成 0/0 个待办，专注 0 分钟，最长空白 2 小时。

数据来源：

- `records`
- `todos`
- `tracker_logs`
- `focus_sessions`
- `expenses`
- `body_logs`

目标：让用户一眼知道今天有没有被记录下来。

#### 2. 今日节奏

做一条横向 24 小时节奏条，表达记录密度和空白。

显示规则：

- 有记录的时间段点亮。
- 记录密集的区域更亮或更厚。
- 长空白区域用更长的淡色段表示。
- 点击某个时间段可以跳到「线」页对应位置，后续实现。

核心指标：

- 第一条记录时间
- 最后一条记录时间
- 最长空白时长
- 记录最密集时段

这块是「盘」页最有 Dayline 特色的部分。

#### 3. 今天由什么组成

替换现在的“分类统计”列表。

展示方式：

- 用轻量横条或胶囊组展示真实占比。
- 不再使用伪数据。
- 不做很重的报表感。

分类来源：

- 记录 tags
- parser 推断类型
- tracker 名称
- todo、focus、expense、body 等结构化数据

示例：

- 运动 4
- 情绪 4
- 学习 2
- 工作 1
- 生活 1

#### 4. 今日洞察

只展示 1-3 条真正有用的判断。

示例：

- “今天最密集的记录出现在 15:00-17:00。”
- “运动相关内容最多，占今天记录的 33%。”
- “上午记录较少，最长空白出现在 09:20-12:10。”
- “今天没有待办压力，适合做轻量复盘。”

规则：

- 没有足够数据时不硬编。
- 不显示固定假 AI 文案。
- 第一阶段可以用规则生成，后续再接 AI。

#### 5. 晚间复盘

这是展开态的核心输入区域。

保留三问，但必须支持真实填写和保存：

- 今天值得保留的是
- 今天可以调整的是
- 明天最小行动是

保存策略：

- 建议新增 `daily_reviews` 表保存结构化复盘。
- 或短期用 `records` 表保存 `type = review` 的记录。
- 长期推荐独立表，方便 Markdown 和 AI 使用。

建议字段：

- `date`
- `kept`
- `adjust`
- `next_action`
- `created_at`
- `updated_at`

#### 6. 生成笔记

主按钮：生成今日 Markdown

状态：

- 未复盘：按钮文案为“生成草稿”
- 已填写复盘：按钮文案为“生成今日笔记”
- 已生成：显示文件名和“重新生成”

导出 Markdown/JSON 不再作为大按钮铺在页面底部。JSON 可以收进更多菜单，Markdown 是主路径。

## 6. Markdown 个人数据库方案

### 文件夹结构

默认文件夹：

```text
DaylineNotes/
  2026/
    05/
      2026-05-13.md
```

后续可在设置中选择根目录。

### 文件命名

支持三种命名方式：

1. 默认日期

```text
2026-05-13.md
```

2. 日期 + 默认标题

```text
2026-05-13_复盘.md
```

3. 日期 + 自定义标题

```text
2026-05-13_今天状态很好.md
```

建议默认使用第一种，保证“一天一篇”稳定沉淀。

### 冲突策略

默认同一天生成同一个文件。

- 第一次生成：创建文件。
- 再次生成：更新同名文件。
- 用户自定义标题：生成新文件。
- 如果担心覆盖，后续可以加“保留历史版本”设置。

### Markdown 模板

```md
---
date: 2026-05-13
title: 2026-05-13
source: dayline
version: 1
generated_at: 2026-05-13T21:30:00+08:00
record_count: 12
todo_completed: 0
todo_total: 0
focus_minutes: 0
tags:
  - 运动
  - 学习
  - 情绪
---

# 2026-05-13 复盘

## 今日概览

今天留下了 12 条记录，完成 0/0 个待办，专注 0 分钟，最长空白 2 小时。

## 今日节奏

- 第一条记录：08:42
- 最后一条记录：21:10
- 最密集时段：15:00-17:00
- 最长空白：09:20-12:10

## 时间线

- 08:42 跑步
- 10:15 工作
- 15:20 学习

## 今天由什么组成

- 运动：4
- 情绪：4
- 学习：2
- 工作：1
- 生活：1

## 今日洞察

- 今天运动相关内容最多。
- 下午记录更密集。
- 上午存在较长空白。

## 晚间复盘

### 今天值得保留的是

...

### 今天可以调整的是

...

### 明天最小行动是

...

## 原始记录索引

本节保留给未来 AI 检索与结构化分析。
```

### 面向未来 AI 的要求

Markdown 必须保持长期稳定：

- 固定 front matter 字段。
- 固定一级/二级标题。
- 时间线保留原始事实。
- 洞察和复盘分开。
- 不把 AI 生成内容和用户原始记录混在一起。

后续接 AI 时可以基于这些 Markdown 做：

- 周报
- 月报
- 情绪趋势
- 习惯变化
- 个人知识库问答
- “我最近为什么累”的长期模式分析

## 7. 数据同步方案

「盘」页不应维护另一套记录数据，而是读取「记」页已经写入的本地 SQLite。

### 已有数据源

- `records`：普通记录、情绪、睡眠等
- `todos`：待办
- `trackers`：打卡项目
- `tracker_logs`：打卡记录
- `focus_sessions`：专注
- `expenses`：消费
- `body_logs`：身体数据

### 新增数据源

建议新增：

- `daily_reviews`：晚间三问
- 可选 `generated_notes`：记录已生成的 Markdown 文件路径和生成时间

### 同步规则

- 用户在「记」页新增内容后，「盘」页自动刷新。
- 用户在「线」页编辑记录后，「盘」页自动刷新。
- 用户在「盘」页填写复盘后，Markdown 生成使用最新复盘内容。
- Markdown 生成前重新读取数据库，避免用旧数据。

## 8. 交互与动效方案

### 复盘球动效

目标是“顺滑、安静、有仪式感”，不是花哨。

动效建议：

- 呼吸：轻微 scale + 光晕透明度变化。
- 打开：球体放大，背景轻微变暗，内容从球心向外展开。
- 收起：内容回收到球体，球体恢复呼吸。
- 晚上 6 点后：光晕更明显，但不闪烁。
- 已完成复盘：光晕稳定，颜色更温和。

技术建议：

- Flutter 先用 `AnimationController` + `TweenSequence` 实现。
- 不引入复杂 3D 引擎。
- 使用现有颜色系统，避免大面积紫蓝渐变。
- 保证低端机不掉帧：少用重阴影、模糊和大面积 shader。

### 页面展开方式

建议第一阶段采用页面内展开，而不是弹窗。

原因：

- 更稳定。
- 更容易适配键盘。
- 更容易被测试。
- 不会和底部导航冲突。

后续可以升级为半屏/全屏沉浸式展开。

## 9. 空状态与边界状态

### 今天没有记录

收起态文案：

> 今天还没有内容。先去「记」里留下第一句话。

展开态只显示：

- 空节奏条
- 引导记录按钮
- 不显示假总结

### 数据很少

少于 3 条记录时：

- 不显示复杂洞察。
- 只显示“今天已经开始被记录”。
- 允许生成草稿 Markdown。

### 晚上前

18:00 前：

- 球体保持普通呼吸态。
- 文案偏轻：“今天还在进行中。”
- 仍允许手动打开。

### 晚上后

18:00 后：

- 球体进入邀请复盘状态。
- 默认展示“打开复盘”。

### 已生成笔记

展示：

- 文件名
- 生成时间
- 重新生成入口

## 10. 实施阶段

### 第 1 阶段：去伪数据，建立真实盘页数据层

目标：

- 移除固定关键词、固定 AI 总结、随机分类统计。
- 建立 `dashboardSummaryProvider`。
- 聚合 records、todos、tracker_logs、focus_sessions、expenses、body_logs。

交付：

- 今日状态真实。
- 分类统计真实。
- 关键词来自 tags 和内容解析。
- 没数据时显示空状态。

### 第 2 阶段：复盘球收起态

目标：

- 重做 `DashboardPage` 首屏。
- 默认展示复盘球。
- 根据时间和数据生成球体状态。

交付：

- 呼吸球组件。
- 18:00 后邀请态。
- 点击打开展开内容。

### 第 3 阶段：展开态信息架构

目标：

- 实现今日状态、今日节奏、今日组成、今日洞察。
- 卡片数量减少，视觉更轻。

交付：

- 24 小时节奏条。
- 最长空白计算。
- 分类胶囊/横条。
- 规则洞察。

### 第 4 阶段：晚间复盘可编辑可保存

目标：

- 三问支持输入、保存、再次编辑。
- 保存后参与 Markdown 生成。

交付：

- `daily_reviews` 数据表或临时记录方案。
- 复盘输入组件。
- 保存成功状态。

### 第 5 阶段：Markdown 生成

目标：

- 一键生成 `.md` 文件。
- 支持文件名模式。
- 放入统一文件夹。

交付：

- `DaylineNotes/YYYY/MM/YYYY-MM-DD.md`
- front matter
- 稳定章节模板
- 重新生成策略

### 第 6 阶段：真机验证和打磨

目标：

- 在真机上验证动效、键盘、滚动、导出权限、文件路径。

交付：

- Widget tests
- 真机截图
- 生成 Markdown 文件检查
- 性能检查，确保动画顺滑

## 11. 验收标准

### 产品验收

- 用户一进「盘」页，不再看到长功能清单。
- 用户能明确知道今天的状态。
- 用户能看到空白时间和记录节奏。
- 用户能完成晚间三问。
- 用户能一键生成当天 Markdown。

### 数据验收

- 页面不再展示伪数据。
- 「记」页新增数据后，「盘」页能同步变化。
- 「线」页编辑后，「盘」页能同步变化。
- Markdown 内容和页面统计一致。

### 体验验收

- 复盘球打开和收起顺滑。
- 18:00 后状态变化自然。
- 键盘输入不遮挡复盘文本框。
- 空状态不尴尬，不硬编 AI 总结。

### 工程验收

- 数据聚合逻辑有测试。
- Markdown 生成逻辑有测试。
- 页面关键组件有 widget test。
- 真机安装后功能可用。

## 12. 暂不做的内容

第一阶段不做：

- 真 AI 总结。
- 云同步。
- 多设备同步。
- 复杂图表。
- 复杂 3D 球体。
- 自动生成周报/月报。

这些可以在 Markdown 数据库稳定后再做。

## 13. 最终方向

「盘」页最终应该成为 Dayline 的收束点：

- 白天，用户用「记」低成本留下生活碎片。
- 过程中，用户用「线」看事实和空白。
- 晚上，用户打开「盘」里的复盘球，把一天整理成判断。
- 最后，一键生成 Markdown，让这一天进入个人数据库。

一句话概括：

> 「记」负责捕捉，「线」负责呈现，「盘」负责沉淀。
