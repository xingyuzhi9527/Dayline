## 主要改动

### 1. `线` 页：改成双侧记录时间线

改动内容：

### 2. `线` 页：按时间间隔显示不同空白长度

新增/调整逻辑：

- 记录之间间隔越久，中间留白越长。
- 目前分档大致为：
  - 10 分钟以内：短间隔
  - 30 分钟以内：稍长
  - 60 分钟以内：中等
  - 2 小时以内：明显拉开
  - 4 小时以内：更长空白
  - 4 小时以上：最长空白

这样用户能通过线的长度感知“这段时间发生了什么”和“哪里是空白”。

后续可以把这套空白判断迁移到 `盘` 页的 24 小时节奏条里。

### 6. `记` 页：文本输入与键盘联动

涉及文件：

- `lib/features/flash_record/flash_record_page.dart`
- `test/widget_test.dart`

用户反馈：

- 文本框被键盘压住。
- 输入法弹出速度快，文本框展开动画被挤压。
- 多次点击键盘和空白处有卡顿。
- 点击键盘按钮偶发无响应。

当前方案：

- 小胶囊先被键盘顶上去。
- 键盘稳定后再展开输入框。
- 减少键盘启动期间的额外动画。
- 输入展开使用更柔和的 curve 和内容淡入/缩放。
- 日志 tag 使用 `DaylineInput` / `DaylineKB` 相关输出，方便真机重现。

这部分仍属于可继续打磨区域，尤其是低端机或不同输入法下的动画流畅度。

## 已知问题与注意事项

### 1. 文本输入动画仍可继续优化

虽然已经改成“两阶段”：

1. 胶囊先被输入法顶上去。
2. 键盘停止后再展开。

但不同输入法、不同系统动画速度下，仍可能出现轻微卡顿。后续应在真机上录屏对比，并考虑进一步降低布局重算。

### 2. `盘` 页当前仍是旧实现

今天完成的是 `盘` 页重构计划书，还没有开始实现复盘球。

当前 `DashboardPage` 仍存在：

- 固定关键词。
- 固定 AI 总结。
- 分类统计伪数据。
- 导出入口视觉权重过高。

下一阶段应优先去伪数据。

### 3. 模型包走 Git LFS

新环境 clone 项目后，需要确保 Git LFS 正常：

```bash
git lfs pull
```

否则 `assets/stt/sense_voice_small_zh.tar.bz2` 可能只是 LFS 指针文件。

### 4. 部分文件存在历史 mojibake 文本

项目中有一些中文字符串曾出现编码显示异常。当前测试能通过，但后续整理 UI 文案时建议统一清理中文字符串。
