## 1. 背景

本轮围绕三个方向推进：

- **新模块落地**：文档库和片刻图片是路线图里计划已久的能力，这次完成 MVP 接入。
- **构建工程化**：双账户协作签名问题一直是个痛点，需要一次性配置好 release 签名并解决国内 Gradle 下载依赖失败的问题。
- **闪记体验重构**：闪记页是整个 App 最高频的入口，上一轮虽然收敛了解析器和语音确认流程，但页面结构本身越来越臃肿，需要做一次结构性重写。

此外仪表盘、时间线、Markdown 存储和数据库层也做了配套增强。

---

## 2. 构建工程化

### 2.1 Release 签名

- 生成 release keystore，路径 `E:\build.gradle.keystore`，别名 `liflow-release`，有效期 10000 天。
- `android/app/build.gradle.kts` 改为从 `android/key.properties` 读取签名配置，找不到时自动降级为 debug 签名。
- `android/key.properties` 加入 `.gitignore`，密钥密码不进入版本控制。
- 构建产物使用 V2 签名（APK Signature Scheme v2）。

双账户只需要各自创建 `key.properties` 指向同一个 keystore 文件，构建出的 APK 签名即自动一致。

### 2.2 国内 Gradle 构建

- `android/build.gradle.kts` 和 `android/settings.gradle.kts` 新增阿里云 Maven 镜像（google、public、gradle-plugin）。
- 原 `google()` / `mavenCentral()` 保留作为 fallback。

解决了 Gradle 直连 Google Maven 时 TLS 握手被阻断的问题。

## 5. 闪记页重构

### 改动范围

`lib/features/flash_record/flash_record_page.dart`（+412 行）和通知器 `flash_record_notifier.dart`。

### 主要变化

- 页面拆分为更清晰的状态区：输入态、编辑态、提交中。
- 类型切换和标签修改集中在编辑卡片区域。
- 语音识别结果直接进入编辑卡片，不再经过独立的确认步骤。
- 文字输入同样可以预览、修改后重新解析再提交。
- 解析器自动化进一步降噪，减少误判。
